AI足球赛事研判指挥舱 HTML授权版 使用说明

一、访问与授权
1. 客户打开卖家提供的网页地址。
2. 页面显示设备码，客户把设备码发给卖家。
3. 卖家生成授权码后，客户输入授权码完成激活。
4. 激活状态保存在当前浏览器；期限授权按首次激活时间计算，永久授权长期有效。

二、赛事数据
1. 页面自动读取500彩票网最新竞彩足球，客户不需要下载数据。
2. GitHub Actions每天北京时间08:15、14:15、18:15自动更新最新竞彩和最近赛果。
3. 更新成功才覆盖赛事；遇到403、网络错误或解析失败时保留上一版，避免客户界面被清空。
4. data/jc_history.json保留约400天压缩赛果，维护721支以上球队近期画像；Interwetten五年历史画像作为长期联赛基线。
5. 页面顶部会显示“500数据已更新”“当前无竞彩”或“更新失败保留旧版”。

三、手动录入
1. 点击“手动录入赛事”。
2. 只填写主队、客队和比赛时间，系统自动匹配球队与联赛历史画像。
3. 手动赛事保存在客户当前浏览器，不会写回公共仓库。
4. 点击“删除当前手动赛事”可清理该场，不影响历史画像和500日常数据。
5. 手动赛事没有实时赔率时，系统会明确标注资料有限并降低信心，这属于正常数据风控。

四、核心功能
1. 赛事雷达：按球队、联赛、场次和风险筛选比赛。
2. 生成全部研判：按信心和冷门压力汇总全部比赛。
3. 一眼看懂：集中展示主方向、次方向、执行口径、比分、大小球和阵容影响。
4. 单场报告：展示统一结论、胜平负概率、两个最得意比分；开放局额外给出两个大球冷门比分。
5. 球员情报：展示500预计首发、伤病、停赛和近21天公开新闻；预计阵容不等同于官方首发。
6. 多Agent圆桌：数据底座、近期状态、阵容伤停、实时新闻、联赛画像、赔率、爆冷、反防平、比分脚本和圆桌仲裁全员执行。
7. 历史回看：右上角查看此前研判、最终赛果和方向命中情况，可按球队、联赛或日期搜索。

五、注意事项
1. 客户公开仓库需要上传index.html、data、tools、.github、.nojekyll、说明和免责声明。
2. 不要把seller_tools或授权码生成器上传到公开仓库。
3. HTML静态授权用于普通交付管理，不等于强DRM。
4. 换浏览器、清空浏览器数据、换电脑或设备码变化时，需要重新获取授权码。
5. 演示网址后加?demo=1；演示模式只显示内置样例。
